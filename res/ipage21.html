git简要

•••创建新仓库
创建新文件夹,打开,然后执行--->git init <---以创建新的git仓库.


•••克隆仓库
执行如下命令以创建一个本地仓库的克隆版本--->git clone /path/to/repository <---
如果是远端服务器上的仓库,执行如下命令将在本地创建一个本地克隆版--->git clone username@host:/path/to/repository <---


•••工作流
你的本地仓库由git维护的三棵"树"组成.第一个是你的(工作目录),它持有实际文件;
-----------------------------|-------------------------------------------------
----------------------------add------------------------------------------------
-----------------------------|-------------------------------------------------
第二个是(缓存区),它像个缓存区域,临时保存你的改动;
-----------------------------|-------------------------------------------------
----------------------------commit----------------------------------------------
-----------------------------|-------------------------------------------------
最后是(head),指向你最近一次提交的结果.


•••添加与提交
添加是将文件改动的文件添加到缓存区,使用--->git add filename <---添加指定文件名的文件;或者通过--->git add * <---添加所有文件.这是git基本工作流程的第一步.
提交是将缓存区的内容提交到head.使用命令--->git commit -m "代码提交信息" <---,但是还没有提交的远端仓库.


•••推送改动
你的改动现在已经在本地仓库的head中了,执行命令--->git push origin master <---将改动提交到远端仓库.(可以把master换成你想要推送的任何分支).
如果你还没有克隆现有仓库,并欲将你的仓库连接到某个远程服务器,你可以使用命令--->git remote add origin [server] <---添加.这样你就能够将你的改动推送到所添加的服务器上了.


•••分支
分支是用来将特性开发绝缘开来的.在你创建仓库的时候,master是默认的.在其他分支上进行开发,完成后再将它们合并到主分支上.
                      feature_x
               /--------------------\
             /             master        \
---------/---------------------------\------------------>
        (branch)                              (merge)
            创建一个叫做feature_x的分支,并切换过去是用命令--->git checkout -b feature_x<---
切换回主分支使用命令--->git checkout master<---
在把新建的分支删掉,使用命令--->git branch -d feature_x<---
除非你将分支推送到远端仓库,不然该分支别人是看不到的,使用命令--->git push origin [branch]<---


•••更新与合并
更新你的本地仓库至最新改动,执行命令--->git pull<---拉取远端仓库新版本,以便在你的工作目录中获取(fetch)并合并(merge)远端的改动.
使用命令--->git merge [branch]<---合并其他分支到你的当前分支.
一般git都会尝试去自动合并改动.不幸的是,自动合并并非次次都能成功,并可能导致冲突(conflicts).这时候就需要你修改这些文件来人肉合并这些冲突了.改完之后,你需要执行--->git add filename<---命令将它们标记为合并成功.
在合并改动之前,也可是使用命令--->git diff [source_branch] [target_branch]<---来查看.


•••标签
在软件发布时创建标签是被推荐的.这是个旧有概念,在SVN中也有.可以执行如下命令创建一个1.0.0的标签--->git tag 1.0.0 1b2e1<---
上面的1b2e1是你想要标记提交ID的前5为字符,可以使用命令--->git log<---获取提交ID.
